# 004 - 콘텐츠 / 메시지 라우터 테스트 작성하기

투두 2번 섹션 — `/v1/contents`, `/v1/messages` (GET/POST) 라우터에 **20개 테스트** 를 추가했어요.
그 과정에서 두 가지를 발견했는데, 하나는 의존성 업그레이드로 인한 **회귀**, 또 하나는 **라우터 로직 버그** 였어요.

## 1. 콘텐츠 라우터 (`GET /v1/contents`)

이 라우터는 `store/users.csv` 와 `store/contents.csv` 를 polars 로 직접 읽어서, 키워드 매칭과
relevance 정렬을 해요. 외부 의존성으로 `googletrans` 도 부르고 있죠.

### 1-1. 외부 의존성 설계

세 가지를 격리했어요.

- **CSV 파일**: 002 작업에서 만든 `tmp_store` + `csv_writer_helper` 픽스처로 임시 디렉터리에
  헤더와 행을 넣어 격리.
- **googletrans**: `mocker.patch("app.api.views.contents.translate_keywords", return_value=[])` 로 봉쇄.
  실제 구글 번역 API를 치지 않게.
- **slack_app.client**: 콘텐츠 라우터에는 영향이 없어 무시.

### 1-2. 구버전 polars 회귀 발견

`uv sync` 후 polars 가 1.40.1 로 업그레이드됐는데, 라우터 코드는 polars 0.x API를 사용하고 있었어요.

- `pl.col(...).apply(...)` → 1.x 에서 제거됨 (`map_elements` 로 대체)
- `df.groupby(...)` → 1.x 에서 제거됨 (`group_by` 로 대체)
- `pl.count()` → 1.x 에서 deprecated (`pl.len()` 로 대체)

런타임에 `AttributeError` 가 터질 텐데, 운영 환경의 `requirements.txt` 가 `polars==0.19.18` 이라
지금까지는 멀쩡했던 거예요. **버그 수정 정책에 따라 코드는 손대지 않고**, `polars>=0.19,<1.0` 로
버전 범위를 좁혔어요. 이러면 운영과 동일한 동작이 보장돼요.

### 1-3. `category` 필터 동작 안 함 → **수정 완료**

테스트를 쓰다가 `category` 쿼리 파라미터가 결과를 전혀 좁히지 못한다는 사실을 발견했어요.

#### 원래 코드 (버그)

```python
# 유니크한 콘텐츠만 가져오기
joined_df = contents_df.unique(subset=["content_url"]).join(
    users_df, on="user_id", how="inner"
)

if keyword == "전체보기":
    ...

if category:
    contents_df = contents_df.filter(contents_df["category"] == category)  # ← (1)

# ... 키워드 매칭 ...
matched_dfs = [
    joined_df.filter(joined_df.apply(...))  # ← (2): joined_df 를 사용. (1) 의 결과를 안 씀!
    for keyword in set(keywords)
]
```

`(1)` 에서 `contents_df` 만 필터링하고, 정작 매칭은 `(2)` 의 `joined_df`(필터 전 join 결과)에서 일어나요.
결과적으로 `category=기술 & 언어` 를 줘도 `일상 & 생각 & 회고` 글이 그대로 응답에 섞여 들어왔어요.
또한 `if keyword == "전체보기":` 분기는 아예 `if category:` 보다 앞에 있어서, 전체보기 모드에서는
원래도 카테고리 필터가 무시되는 구조였어요.

#### 수정한 코드

사용자가 직접 수정 요청을 줘서 다음과 같이 고쳤어요.

```python
joined_df = contents_df.unique(subset=["content_url"]).join(
    users_df, on="user_id", how="inner"
)

# 카테고리 필터는 전체보기와 키워드 검색 양쪽 모두에 적용한다.
if category:
    joined_df = joined_df.filter(joined_df["category"] == category)

if keyword == "전체보기":
    contents = joined_df.sort(["dt"], descending=descending)
    ...
```

핵심 두 가지:

1. 필터링 대상을 `contents_df` 가 아니라 **실제 매칭이 일어나는 `joined_df`** 로 바꿨어요.
2. 필터 분기를 `if keyword == "전체보기":` **앞으로** 옮겨서 전체보기 모드에서도 카테고리가 적용되게
   했어요.

#### 테스트도 정상 동작 기준으로 + 케이스 추가

기존에 "현재 (잘못된) 동작 기록" 으로 작성했던 `_currently_does_not_filter` 테스트는 이름과 단언을
바꿔서 정상 동작을 검증하도록 했어요. 거기에 더해 **전체보기 + category** 케이스를 새로 한 개 추가했어요.

| 케이스                                | Before (버그)          | After (정상)             |
| ------------------------------------- | --------------------- | ------------------------ |
| 키워드 검색 + category="기술 & 언어"   | 두 글 모두 포함       | 기술 글만 포함           |
| 전체보기 + category="기술 & 언어"     | (테스트 없었음)       | 기술 카테고리 글만 포함  |

### 1-4. 통과한 케이스 10개

- ✅ 전체보기 + dt 내림차순 + limit/offset
- 🌀 큰 offset → 빈 data
- 🌀 descending=False → 오름차순
- ✅ 키워드 매칭 + relevance
- 🌀 매칭 없음 → 빈 결과
- 🌀 콤마/슬래시로 분리된 키워드 → 모두 매칭
- ✅ category 필터 (키워드 검색)
- ✅ category 필터 (전체보기)
- ✅ job_category 필터
- ⚠️ limit > 50 → 422
- ⚠️ keyword 누락 → 422

## 2. 메시지 라우터 (`GET / POST /v1/messages`)

이 라우터는 슬랙 메시지를 조회/수정해요. 외부 의존성은 슬랙 API 하나뿐이라 콘텐츠 라우터보다 단순해요.

### 2-1. `slack_app.client` 다루기

`/v1/messages` 라우터는 `app.api.views.contents.slack_app.client` 를 호출해요. 이 객체는
`app.slack.event_handler.app.client` 와 동일한 싱글톤이라, 메서드만 mock 으로 갈아끼우면 돼요.

```python
@pytest.fixture
def slack_client(mocker):
    client = contents_view.slack_app.client
    mocker.patch.object(client, "conversations_history")
    mocker.patch.object(client, "conversations_replies")
    mocker.patch.object(client, "chat_update")
    mocker.patch.object(client, "chat_getPermalink")
    return client
```

각 테스트는 필요한 메서드의 `return_value` 또는 `side_effect` 만 설정하면 돼요. 슬랙 서버가 살아있을
필요가 전혀 없죠.

### 2-2. 통과한 케이스 10개

GET (7개):
- ✅ admin → 단일 메시지
- ✅ multiple_messages=True → 리스트
- ✅ type=reply → conversations_replies 호출 분기
- ⚠️ 비-admin → 403
- 🌀 인증 누락 → 403
- ⚠️ ts 매칭 안 되는 경우 → 404
- ⚠️ SlackApiError → 409

POST (3개):
- ✅ admin → chat_update + permalink
- ⚠️ 비-admin → 403
- ⚠️ SlackApiError → 409
